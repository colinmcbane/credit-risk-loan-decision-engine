# Credit Risk & Loan Decision Engine

An end-to-end machine learning pipeline that scores SBA 7(a) loan applications, tests for regulatory fairness, and generates ECOA-compliant adverse action letters using AI — deployed as an interactive web dashboard with a live loan application form.

**🔗 Live Demo:** [credit-risk-loan-decision-engine.onrender.com](https://credit-risk-loan-decision-engine.onrender.com)

*Note: The free hosting tier spins down with inactivity — first load may take 30–50 seconds.*
*Email delivery is available when running locally. The deployed version displays generated letters on-screen due to Render free tier SMTP restrictions.*

---

## What It Does

This project simulates the full lifecycle of a production credit risk system at a regulated financial institution:

1. **Scores loans** using a LightGBM champion model trained on 382,144 historical SBA 7(a) loans
2. **Explains decisions** with SHAP-derived reason codes for every denial
3. **Tests for bias** across geography, industry, business age, and loan size per ECOA Regulation B
4. **Generates adverse action letters** via the Gemini API, meeting all ECOA notice requirements
5. **Delivers letters by email** through Gmail SMTP automation
6. **Serves everything live** through a six-tab Dash dashboard where anyone can submit a loan application and get scored in real time

## Key Results

| Metric | Industry Standard | This Model |
|---|---|---|
| AUC | 0.70 – 0.85 | **0.9667** |
| KS Statistic | 30 – 50 | **83.63** |
| Gini Coefficient | 40 – 70% | **93.34%** |

Champion model selected through a champion-challenger framework: **LightGBM** vs. XGBoost vs. Logistic Regression, validated with 5-fold stratified cross-validation and SMOTE for class imbalance.

## Pipeline Architecture

| Phase | Description | Key Output |
|---|---|---|
| 1 — SQL Cleaning | Ingestion and cleaning of raw SBA FOIA data using SQLite | 382,144 clean loan records |
| 2 — EDA | Exploratory analysis in R (Quarto) and Python | Distribution analysis, correlation matrix, default trends |
| 3 — Feature Engineering | One-hot encoding, log transforms, z-score scaling | 26-feature matrix, persisted scaler parameters |
| 4 — Modeling | Champion-challenger framework with SHAP explainability | LightGBM champion (AUC 0.9667) |
| 5 — Fairness Analysis | ECOA disparate impact testing (4/5ths rule, equalized odds) | SR 11-7 model card, fairness audit across 4 dimensions |
| 6 — AI Decision Engine | SHAP reason codes → Gemini letters → Gmail delivery | ECOA-compliant letters for 1,051 flagged loans |
| 7 — Dashboard | Six-tab interactive Dash app with live scoring form | Deployed web application |

## Out-of-Sample Validation

The model was stress-tested against **200 synthetic loan applicants generated by four different AI systems** (Gemini, ChatGPT, Claude, Copilot), each with intentionally designed low / moderate / high risk tiers.

Key finding: given identical prompts, the four AI sources produced meaningfully different risk distributions — Gemini generated the most extreme profiles (88% denial rate) while Claude generated the most conservative (16%). This variance analysis is visualized in the dashboard's Synthetic Analysis tab.

This process also surfaced and fixed a real production bug: raw synthetic inputs initially bypassed the Phase 3 scaling pipeline, producing uniformly low risk scores. The fix — a centralized `preprocess_and_scale()` utility applied to all inference paths — mirrors how scaling skew is handled in real MLOps systems.

## Regulatory Compliance

- **ECOA Regulation B (12 CFR Part 1002)** — every adverse action letter includes specific reasons, a statement of applicant rights, free credit report disclosure, and CFPB contact information
- **SR 11-7 Model Risk Management** — full model card with documented limitations: SMOTE probability shift, tree-model tail-risk plateaus, excluded features, and geographic scope constraints
- **SHAP reason codes** — denial reasons are derived from each applicant's top positive SHAP contributions, not heuristics

## Tech Stack

**Modeling:** Python, LightGBM, XGBoost, scikit-learn, SHAP, imbalanced-learn (SMOTE), Fairlearn
**Data:** SQLite, pandas, NumPy, R (Quarto) for EDA
**AI Integration:** Google Gemini API (letter generation)
**Delivery:** Gmail SMTP automation
**Dashboard:** Plotly Dash, dash-bootstrap-components
**Deployment:** Render, gunicorn

## Running Locally

```bash
# Clone and install
git clone https://github.com/colinmcbane/credit-risk-loan-decision-engine.git
cd credit-risk-loan-decision-engine
pip3 install -r requirements_dashboard.txt

# Configure credentials
cp .env.example .env
# Fill in GEMINI_API_KEY, GMAIL_ADDRESS, GMAIL_APP_PASSWORD

# Run the dashboard
python3 phase7_dashboard/app.py
# Open http://127.0.0.1:8050
```

To run the full Phase 6 adverse action pipeline:

```bash
python3 phase6_ai_decision_engine.py   # DRY_RUN=True by default
python3 validate_phase6.py             # verify outputs
```
